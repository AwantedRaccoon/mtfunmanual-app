# Unmanual 前端系统（内部迭代 V2.5）

## 定位

V2.5 是当前前端设计迭代的内部代号，不是 App 版本。App 首发版本为 1.0；用户可见页面、状态和辅助功能文案不显示“V2.5”。该前端服务于需要快速确认今天、回看个人记录和准备下一步的人；每个页面只完成一个主要任务，不把 App 扩展成医疗 Dashboard。

## 视觉公式

> 时间刻度作为首页签名；其余页面使用同一套纸本台账语法，而不复制首页刻度。

### 色彩

不新增色值，只使用现有语义令牌：

| 角色 | 令牌 | 色值 |
| --- | --- | --- |
| 外层纸面 | `rice` | `#EFE6D4` |
| 可编辑纸张 | `paper` | `#F7F0E3` |
| 结构与主行动 | `indigo` | `#172F4A` |
| 深色收尾 | `indigoDeep` | `#0F2135` |
| 判断与索引 | `vermilion` | `#B94D42` |
| 下一事项与医疗信息面 | `blue` | `#477594` |

每屏只允许一个浅蓝重点面和一个深靛主行动／收尾面。芥末金只保留给底部当前标签的定位线，不进入内容卡片。

### 字体

- 展示字：`Songti SC`，用于页面题名、关键数字和记录正文；
- 正文字：系统无衬线，用于说明、输入和操作；
- 功能字：`Arial Narrow`，用于日期、版本、状态和英文眉题。

### 构图

```text
页面题名                         LOCAL / SECTION
一句当前任务说明                         状态
──────────────────────────────────────

浅蓝：当下最需要注意的一件事
纸白：可以查看或编辑的原始事实
深靛：主要行动或本页收尾
──────────────────────────────────────
本地保存／系统备份边界
```

- 主页面是单列台账；规则线负责分组，不使用一组漂浮圆角卡片。
- 旅程用真实日期和间隔形成时间路书；方案用真实版本号形成档案；档案页的数量来自本机真实记录。
- 编辑页使用持续可见标签、矩形纸白输入面和底部唯一保存动作。
- “本地保存”只说明 App 的处理与存储方式：App 不主动上传或实时同步；iOS 仍可能按用户设置将 App 数据纳入系统备份。不得再用“仅本机”暗示系统备份不存在。

## 页面映射

| 页面 | 单一任务 | 签名结构 |
| --- | --- | --- |
| 今天 | 看清此刻并完成一次记录 | HRT 日数刻度 |
| 旅程 | 沿时间路线回看主动留下的事实 | 此刻起点 + 方案路标 + 真实日期间隔 |
| 方案 | 确认当前版本并记录变化 | 当前档案 + 版本台账 |
| 档案 | 核对并带走自己的数据 | 本机档案封套 + 导出工作台 |
| 编辑页 | 完成一次明确输入 | 字段纸张 + 固定保存区 |

### 旅程页正式方案：时间路书

- 页面从“此刻”开始，记录按时间向过去展开；朱红路线是全页唯一视觉主张。
- 日期间的路线留白来自真实日历间隔，不把无记录的日子解释成漏签或失败。
- 方案只在上下文变化处作为路标出现，记录正文仍保留与原方案版本的关联。
- 最近一条记录使用浅蓝面，其余条目保持开放纸面，不恢复为卡片列表。
- 辅助功能字号下取消固定路线列，改为纵向元数据与正文，避免日期和方案信息被压窄。

## 自我审查

- 保留温暖纸色并不是为了制造通用“复古感”，而是因为它已是 MTF不全书的品牌材料；真正区分 V2.5 的是时间刻度、时间路书和版本档案都对应真实数据。
- 不让每页都使用深色大横幅；深靛只承担行动和收尾，避免旧版页面头重脚轻。
- 不为现代感引入渐变、玻璃、Bento 或圆角药丸；密度通过字号、规则线和色面层级控制。

## 兼容与边界

- iOS 17 起，支持动态字体、减少动态效果、VoiceOver 标签和安全区；
- Accessibility 字号下，底部四项导航改为明确的 2×2 布局并保留完整文字、选中状态与 44 pt 命中区。正文继续响应 Accessibility 5；持久导航视觉字号最高使用 Accessibility 2，避免在 320×568 上裁切或吞没内容，VoiceOver 标签不受视觉上限影响；
- 该设计系统是唯一运行时前端，不保留旧首页、旅程或方案实验的启动参数；`V2.5` 只用于内部设计追踪；
- 调试入口只服务正式页面的空数据、演示数据和编辑流程验证；
- 此轮不修改 SwiftData 模型、医疗规则或上传行为。
