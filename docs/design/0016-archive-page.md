# V2.5 档案页

## 页面任务

第四个主页面不再承担“方案匹配 + 工具集合”。它只回答两个问题：这台设备里已经留下了什么，以及用户怎样核对、带走或删除这些内容。

方案匹配从产品范围中取消。单位换算和知识搜索仍可存在，但降为档案底部的“随身附页”，不再决定页面身份。

## 视觉主张

唯一的签名结构是一份带深靛装订脊和芥末金错位底版的“本机档案封套”。装订脊上的 `LOCAL / 04` 同时表达数据位置和第四导航位置，不是装饰编号。封套内只显示真实数量与日期范围，不做健康评分、依从率或伪造的完整度百分比。

下方内容恢复为紧凑台账。深靛块只给“整理就诊材料”这个主动作；原始数据、隐私和附页逐级降低视觉权重，避免重新变成一组同权工具卡片。

## 交互边界

- 就诊摘要、原始数据、本地存储、删除、单位换算和知识搜索入口先提供结构预览；文件生成、删除和换算规则属于后续数据层任务。
- 导出前必须完整预览范围；导出文件离开 App 后不再宣称受 App 保护。
- 删除入口不使用“一键无痕”措辞，必须列出准确对象、关联影响和确认步骤。
- 页面数量来自本机 SwiftData；检查数量按采样日统计，不把同一次检查中的每个指标伪装成多次检查。
- 辅助功能字号下，封套内的数量由横排改为纵排；所有入口保持至少 44 pt 点击区域。
